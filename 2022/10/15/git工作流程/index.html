Git开发流程

一、接手项目

1.新建文件下，在文件下，右键git Bash

2.初始化本地仓库

git init

3.关联远程仓库

git remote add origin https://地址

git remote add origin https://github.com/xxx.git

4.拉取指定分支

git clone -b 分支名 url

git clone -b node https://xxxx

二、开发（简单流程）

直接拿拉取下来的项目分支（拉取的是master的分支）来开发

git add .

git commit -m “第一次提交”

git pull (先 commit 再 pull 最后再push)

git push

三、开发（复杂流程）

1.创建本地分支

checkout -b

2.对本地分支操作

add commit pull push

3.用分支去提交

checkout

本地代码上传github教程 （https）

1.github建仓库

2.进入目录打开git命令行

3.命令提交

ssh（小海龟）

git pull会把本地未提交修改覆盖吗？

没有 add 的文件，不会被覆盖。add 了文件没有 commit，不能 pull。commit 了之后 pull 有冲突，会自行解决或者提示需要编辑。只要 commit 过，都可以用 git reflog 找回。
