巩固ES5,ES6重点知识汇总

Created2022-10-09|Updated2022-10-09
|Post View:

1.两种类型（基本类型、引用类型）

基本数据类型有：number string boolean null underfind

引用类型有：对象

两者的区别：

赋值：基本类型赋值是开辟一个新的地址给其赋值，而引用类型赋值是引用其相同的地址赋值。

比较：基本类型比较的是值是否相等，而引用类型比较的是引用地址是否相等，而不是里面的值（内容）是否真的相等。

函数传参：基本类型传参不影响函数外变量的值，而引用类型传参影响函数外变量的值，因为两者是指向同一个变量。

2.执行环境及其作用域

执行环境分为全局和局部，根据ECMAScript实现所在的宿主环境不同，执行环境的对象也不同。web浏览器中window对象是全局执行环境。全局变量和函数都是作为window对象的属性和方法创建的。某个环境中所有代码执行后，该环境就会销毁，其里面所有定义的变量和函数定义也销毁，执行环境的顺序：每个函数都有自己的执行环境。当执行流进入一个函数，函数的环境就会被推入一个环境栈中。当函数执行完后，栈将其弹出。把控制权还给之前的执行环境。

作用域就是当代码在一个环境中执行的时候，会创建变量对象的一个作用域链。
作用是保证当前执行环境有权访问的所有变量和函数的有序访问。执行环境中，访问的变量顺序是：一层一层往上一级寻找。

3.apply() 和 call() 方法

*apply*:方法*能劫持另外一个对象的方法*，*继承另外一个对象的属性*.

Function.apply(obj,args)方法能接收两个参数
*obj*：这个*对象将代替Function类里this对象*
****args*：这个是*数组****，它将作为参数传给Function

*call:和apply的意思一样*,只不过是*参数列表不一样.*

*obj：这个对象将代替Function类里this对象*
*params*：这个是一个*参数列表*

*1， 每个函数都包含两个非继承而来的方法：call()方法和apply()方法。**
**2，相同点： 这两个方法的作用是一样的。*

一般来说，this总是指向调用某个方法的对象，但是使用call()和apply()方法时，就会改变this的指向。

4.基本包装类型、Function 类型

*Function类型* ECMAScript中没有函数重载的概念，如果同时声明了两个同名函数，而结果则是后面的函数覆盖了前面的函数

ECMAScript中的函数也是对象，因此呢，函数也是有属性和方法的，*每个函数都有两个属性：length和prototype.*，当然，*每个函数也有两个非继承而来的方法 apply()和call()*

5.JavaScript面向对象程序设计

面向对象与面向过程，是解决问题的两种思路

面向过程：先分析问题中的步骤，马上将分出来的步骤封装成函数，再按顺序依次调用即可解决问题

通俗讲就是强调怎么做，先干什么再干什么按顺序来 ，好比我想做饭，我得有菜，我去网上下单买菜取到才能做饭，强调结果

面向对象：先分析问题中的步骤，再将步骤划分不同的不同主体上，最后将顺序调用主体上方法

通俗讲就是强调谁来做，这一步怎么做具体给谁做，比如我想做饭得有菜才行，我首先去网上买菜，我得打开手机下单付款，然后等待平台发货，再交给快递员派送，最后再由我去拿，强调过程对象。

6.原型链和继承之间的关系以及重要作用

和其他的面向对象编程语言不同，最开始js并没有引入class的概念，但是js中有在大量使用对象，为了保证对象之间的联系，JavaScript引入了原型与原型链的概念。

在js中，每一个构造函数都拥有一个prototype属性，这个属性指向一个对象，也就是原型对象。原型对象默认拥有一个constructor属性，指向指向它的那个构造函数，每个对象都拥有一个隐藏的属性[[prototype]]，指向它的原型对象。原型对象就是用来存放实例中共有的那部分属性。js中所有的对象都是由它的原型对象继承而来。而原型对象自身也是一个对象，它也有自己的原型对象，这样层层上溯，就形成了一个类似链表的结构，这就是原型链, 所有原型链的终点都是Object函数的prototype属性.

7.闭包以及this对象指向问题

最简单的闭包就是双层函数嵌套，子函数可以访问使用父函数中的变量，缺点是闭包不会自己清空，会造成内存泄漏，因为因为闭包的原因，子函数访问了父函数中的变量因为父函数return了子函数，函数没执行完，所以不会被清空，变量相当于被保存下来了。

this可以用在构造函数之中，表示实例对象。除此之外，this还可以用在别的场合。但不管是什么场合，this都有一个共同点：它总是返回一个对象指向该作用域本身。

1.箭头函数

刚开始学的时候直接看代码看不懂箭头函数，现在看来箭头函数看着比普通函数简洁的多，代码很少，（）=> {return… } 如果就一个参数前面()可省略，如果return返回就一行代码的话{return}都可以省略直接写那行代码即可

2. 解构赋值，模板字符串

解构赋值语法是一种JavaScript表达式，通过解构赋值，可以将属性/值从对象/数组中取出，赋值给其他变量。
对象和数组逐个对应表达式，或称对象字面量和数组字面量，提供了一种简单的定义一个特定的数据组的方法。

3. 简化对象的写法

对象的属性和值如果是同名可以直接简写写成一个即可，函数也可以直接写进对象里，不用再写Function

4. 函数参数的默认值设置

在 ES6 中定义方法时，我们可以给方法里的参数加一个默认值（缺省值）：

方法被调用时，如果没有给参数赋值，那就是用默认值；

方法被调用时，如果给参数赋值了新的值，那就用新的值。

5. rest参数以及扩展运算符

ES6 引入 rest 参数，用于获取函数的实参，与arguments对象的有所区别

rest参数只包括没有给出名称的参数，而arguments包含所有参数

arguments 对象不是真正的数组，而rest 参数是数组实例（可以用sort, map, forEach, pop等方法）

arguments 对象拥有一些rest参数没有的功能

扩展运算符（…）能将数组转换为逗号分隔的参数序列；好比 rest 参数的逆运算，将一个数组转为用逗号分隔的参数序列，对数组进行解包；

6. Promise

Promise 是一种异步编程的解决方案。

简单来说，Promise是一个容器，里面包含着一个异步操作的结果

从语法上来说，promise(小写的p)是一个对象，从它可以获取异步操作的最终状态

而Promise是一个构造函数，自己身上由reject resolve all方法，原型上有then catch方法

Promise有三种状态，pending 初始态，fulfilled 成功状态，rejected失败状态，只有异步操作的结果可以决定当前是哪一种状态，其他任何操作都无法改变这个状态。

Promise的状态一旦改变，就不会再变，任何时候都可以得到这个结果，状态不可以逆，只能由 pending变成fulfilled或者由pending变成rejected

7.迭代器

迭代器(lterator)是一种接口，为各种不同的数据结构提供统一的访问机制。

任何数据结构只要部署lterator接口，就可以完成遍历操作。

for/of循环，扩展操作符 可以直接操作可迭代对象
